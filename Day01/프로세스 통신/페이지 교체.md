## 1. 페이지 교체란 무엇인가

페이지 교체(Page Replacement)는 RAM이 가득 찬 상태에서 새로운 페이지를 적재해야 할 때
기존에 있는 페이지 중 하나를 선택하여 제거하는 과정이다
가상 메모리 환경에서는 모든 페이지가 항상 RAM에 올라와 있지 않다

---

## 2. 왜 페이지 교체가 필요한가

물리 메모리(RAM)는 용량이 제한적이다
상황

- 새로운 페이지 접근
- 해당 페이지가 RAM에 없음 (Page Fault)
- RAM이 이미 가득 찬 상태
  이때 반드시 어떤 페이지를 제거할지 결정해야 한다
  이 결정이 페이지 교체 알고리즘이다

---

## 3. 페이지 교체 과정

1. 페이지 폴트 발생
2. 빈 프레임이 있는지 확인
3. 없다면 교체 대상 페이지 선택
4. 수정된 페이지라면 디스크에 기록
5. 새로운 페이지를 RAM에 적재
6. 페이지 테이블 갱신
7. 명령어 재실행

## 4. 좋은 페이지 교체 알고리즘의 조건

- 페이지 폴트 최소화
- 오버헤드 최소화
- 지역성(Locality) 반영

---

## 5. 주요 페이지 교체 알고리즘

### 5.1 FIFO (First-In First-Out)

가장 먼저 들어온 페이지를 제거한다.
장점

- 구현 간단
  단점
- 오래 사용되는 중요한 페이지도 제거될 수 있음
- Belady의 이상 현상 발생 가능

### 5.2 Optimal (OPT)

앞으로 가장 오랫동안 사용되지 않을 페이지를 제거한다.
장점

- 페이지 폴트 최소화 (이론적으로 최적)
  단점
- 미래를 알 수 없으므로 실제 구현 불가능
- 비교 기준으로만 사용

### 5.3 LRU (Least Recently Used)

가장 오랫동안 사용되지 않은 페이지 제거
장점

- 지역성 반영
- 성능 우수
  단점
- 구현 비용 큼

### 5.4 LFU (Least Frequently Used)

가장 적게 사용된 페이지 제거
단점

- 오래 전에 많이 사용된 페이지가 계속 유지될 수 있음

### 5.5 Clock 알고리즘

LRU를 단순화한 방식

- 원형 큐 구조
- 참조 비트 사용
- 구현이 비교적 쉬움
  실제 운영체제에서 많이 사용된다.

---

## 6. Belady의 이상 현상

일반적으로 프레임 수가 증가하면 페이지 폴트 수는 감소해야 한다
하지만 FIFO 알고리즘에서는 프레임을 늘렸는데도 페이지 폴트가 증가하는 경우가 있다
이를 Belady의 이상 현상이라고 한다

---

## 7. Thrashing

페이지 교체가 과도하게 발생하여 CPU보다 디스크 작업이 더 많아지는 현상
원인

- 너무 많은 프로세스 실행
- 메모리 부족
  결과
- 성능 급격히 저하
- 시스템이 거의 멈춘 것처럼 보임

---

## 8. 페이지 교체와 지역성

페이지 교체 알고리즘은 프로그램의 지역성(Locality)을 활용한다

- 시간적 지역성
- 공간적 지역성
  LRU는 특히 시간적 지역성을 기반으로 한다

---

## 9. 정리

페이지 교체는

- RAM이 부족할 때
- 어떤 페이지를 제거할지 결정하는 과정

대표 알고리즘

- FIFO
- OPT
- LRU
- Clock
