## 1. 컨텍스트 스위칭이란 무엇인가

컨텍스트 스위칭(Context Switching)은 CPU가 한 프로세스, 스레드에서
다른 프로세스, 스레드로 실행 대상을 변경하는 과정이다.

---

## 2. 컨텍스트(Context)란 무엇인가

컨텍스트란 현재 실행 중인 작업의 상태 정보를 의미한다.

- 프로그램 카운터(PC)
- CPU 레지스터 값
- 스택 포인터
- 프로세스 상태 정보
- 메모리 정보
  작업을 중단했다가 나중에 정확히 이어서 실행하기 위해 필요한 모든 정보이다.

---

## 3. 왜 컨텍스트 스위칭이 필요한가

CPU는 여러 프로세스를 번갈아 실행해야 하며 이 방법을 이용해
멀티태스킹이 가능해진다.

---

## 4. 컨텍스트 스위칭이 발생하는 경우

1. 타이머 인터럽트 발생
2. 입출력 요청으로 대기 상태 전환
3. 우선순위 높은 프로세스 등장
4. 시스템 콜 수행

---

## 5. 컨텍스트 스위칭 과정

### 1단계: 현재 프로세스 상태 저장

- 레지스터 값 저장
- 프로그램 카운터 저장
- PCB에 기록

### 2단계: 다음 프로세스 상태 복원

- 선택된 프로세스의 PCB 확인
- 레지스터 값 복원
- 프로그램 카운터 복원

### 3단계: 실행 재개

CPU는 복원된 상태부터 명령어 실행을 계속한다.

---

## 6. 컨텍스트 스위칭의 비용

컨텍스트 스위칭은 시간이 소요된다.

- 상태 저장 및 복원
- 캐시 무효화 가능성
- 메모리 접근 증가
  이 시간 동안 CPU는 실제 작업을 하지 못하며 이를 오버헤드(Overhead)라고 한다

---

## 7. 프로세스 vs 스레드 컨텍스트 스위칭

### 프로세스 간 전환

- 같은 프로세스 내부
- 메모리 공유
- 상대적으로 비용이 적음
  그래서 스레드는 "경량 프로세스"라고 불린다.

---

## 8. 컨텍스트 스위칭과 스케줄링

운영체제는 스케줄링 알고리즘을 사용해 어떤 프로세스를 실행할지 결정하며
그 결정이 실행되기 위해 컨텍스트 스위칭이 수행된다.
스케줄링 = 누구를 실행할지 결정  
컨텍스트 스위칭 = 실제 전환 과정

---

## 9. 정리

컨텍스트 스위칭은

- 현재 작업 상태 저장
- 다른 작업 상태 복원
- 실행 전환
