## 1. CPU란 무엇인가

CPU (Central Processing Unit)는 컴퓨터에서 명령어를 해석하고
실행하는 핵심 장치이다.
우리가 프로그램을 실행하면 CPU는 메모리에서 명령어를 하나씩 읽어와 처리한다.

---

## 2. CPU의 기본 구성 요소

1. 산술논리연산장치 (ALU)
2. 제어장치 (Control Unit)
3. 레지스터 (Register)

---

## 3. 산술논리연산장치 (ALU)

ALU는 실제 계산을 담당한다.

- 덧셈, 뺄셈, 곱셈, 나눗셈
- AND, OR, NOT 같은 논리 연산
- 비교 연산
  프로그램의 모든 계산은 ALU를 통해 수행된다.

---

## 4. 제어장치 (Control Unit)

제어장치는 CPU의 지휘자 역할을 한다

- 명령어 해석
- 각 장치에 신호 전달
- 실행 순서 전달

---

## 5. 레지스터 (Register)

레지스터는 CPU 내부의 초고속 저장 공간이다.

- 용량이 매우 작음
- 속도는 매우 빠름
- 연산에 필요한 데이터 임시 저장

데표적인 레지스터 종류

- 프로그램 카운터(PC): 다음 실행할 명령어 주소 저장
- 명령어 레지스터(IR): 현재 실행 중인 명령어 저장
- 누산기(Accumulator): 연산 결과 저장

---

## 6. CPU의 동작 원리 (Fetch - Decode - Execute)

CPU는 다음 과정을 반복한다.

### 1) Fetch (인출)

메모리에서 다음 명령어를 가져온다.

### 2) Decode (해독)

명령어를 해석한다.

### 3) Execute (실행)

연산을 수행하거나 작업을 처리한다.

이 과정을 명령어 사이클 (Instruction Cycle) 이라고 하며
CPU는 이 과정을 초당 수십억 번 반복한다.

---

## 7. 클럭(Clock)

CPU는 일정한 박자에 맞춰 동작하며 이 박자를 클럭 (Clock) 이라고 한다.

- GHz 단위로 표현
- 클럭이 높을수록 더 많은 명령어 처리 가능
  단순히 클럭이 높다고 항상 성능이 좋은 것은 아니다.

---

## 8.코어 (Core)와 멀티코어

과거에는 CPU 하나에 코어 하나였다

- 듀얼코어 (2개)
- 쿼드코어 (4개)
- 옥타코어 (8개)

각 코어는 독립적으로 명령어를 처리할 수 있다.
멀티코어 = 동시에 여러 작업 처리 가능

---

## 9. 캐시 (Cache)

CPU와 RAM 사이에 위치한 고속 메모리

- 자주 사용하는 데이터를 임시 저장
- RAM 접근 횟수 감소
- 속도 향상

캐시는 보통

- L1 (가장 빠름)
- L2
- L3
  계층 구조로 구성된다.

---

## 10. CPU와 메모리의 관계

CPU는 직접 RAM에 접근하여 데이터를 가져온다.

1. 명령어 주소 확인
2. 메모리에서 데이터 읽기
3. 레지스터에 저장
4. ALU 연산 수행
   메모리가 느리면 전체 성능이 떨어져서 캐시가 중요하다

---

## 11. CPU와 운영체제의 관계

CPU는 한 번에 하나의 작업만 처리할 수 있다.

운영체제

- CPU 시간을 나누어 여러 프로세스를 실행
- 스케줄링 수행
- 컨텍스트 스위칭 관리
  CPU는 실행 장치이고 운영체제는 CPU 사용을 관리하는 관리자이다.

---

## 12. 정리

CPU는

- 명령어를 읽고
- 해석하고
- 실행하는
  컴퓨터의 핵심 연산 장치이다.

구성:

- ALU -> 계산
- 제어장치 -> 흐름 제어
- 레지스터 -> 임시 저장
