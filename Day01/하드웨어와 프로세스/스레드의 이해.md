## 1. 스레드란 무엇인가

스레드(Thread)는 프로세스 내부에서 실행되는 작업의 흐름이며
프로세스가 하나의 실행 환경이라면 스레드는 그 안에서 실제로 일을 수행하는 단위라고 볼 수 있다.
프로세스 = 실행 환경  
스레드 = 실행 흐름

---

## 2. 왜 스레드가 필요한가

과거에는 하나의 프로세스가 하나의 작업만 수행 했지만 다음과 같은 상황에서는 비효율이 발생한다

- 웹 브라우저가 페이지를 불러오는 동안
- 사용자가 화면을 스크롤하거나 클릭
  이때 하나의 작업이 끝날 때까지 기다린다면  
  프로그램은 멈춘 것처럼 보인다.
  스레드를 사용하면
- 여러 작업을 동시에 수행 가능
- 응답 속도 향상
- 자원 효율 증가

---

## 3. 프로세스와 스레드의 관계

하나의 프로세스는  
하나 이상의 스레드를 가질 수 있다.
프로세스 내부 구조

- 코드 영역 → 공유
- 데이터 영역 → 공유
- 힙 영역 → 공유
- 스택 영역 → 각 스레드마다 별도 존재
  공유: 코드, 데이터, 힙  
  개별: 스택, 레지스터

---

## 4. 스레드의 동작 방식

스레드는 CPU 스케줄링의 대상이 된다
운영체제는

- 여러 스레드를 번갈아 실행
- 컨텍스트 스위칭 수행
  스레드 간 전환은 프로세스 간 전환보다 비용이 적다.
