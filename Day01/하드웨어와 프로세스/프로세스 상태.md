## 1. 프로세스 상태란 무엇인가

프로세스는 실행 중이라고 해서 항상 CPU를 사용하고 있는 것은 아니며
운영체제가 프로세스를 여러 상태로 나누어 관리한다
이를 프로세스 샅애(Process State)라고 한다.

---

## 2. 기본적인 프로세스 상태 5가지

1. 생성(New)
2. 준비(Ready)
3. 실행(Running)
4. 대기(Waiting / Blocked)
5. 종료(Terminated)

---

## 3. 각 상태의 의미

### 3.1 생성 (New)

- 프로세스가 막 만들어진 상태
- 매모리 할당 및 초기화 진행
- PCB 생성
  아직 CPU를 받을 준비가 되지 않은 상태

### 3.2 준비 (Ready)

- CPU를 기다리는 상태
- 실행에 필요한 자원은 모두 준비됨
- Ready Queue에서 대기
  CPU만 할당되면 바로 실행 가능

### 3.3 실행 (Running)

- CPU를 할당받아 명령어를 실행 중인 상태
  CPU는 한 번에 하나의 프로세스만 실행할 수 있다.

### 3.4 대기 (Waiting / Blocked)

- 입출력 작업 등을 기다리는 상태
- CPU를 받을 수 없음

### 3.5 종료 (Terminated)

- 실행이 끝난 상태
- 자원 반환
- PCB 제거

---

## 4. 상태 전이 (State Transition)

프로세스는 한 상태에 머무르지 않고 상황에 따라 다른 상태로 이동한다.

1. New -> Ready
   생성 완료 후 준비 상태로 이동
2. Ready -> Running
   스캐줄러가 CPU를 할당
3. Running -> Waiting
   I/O 요청 등으로 대기
4. Waiting -> Ready
   I/O 완료 후 준비 상태로 복귀
5. Running -> Terminated
   실행 종료

---

## 5. Ready Queue

Ready 상태의 프로세스들은 Ready Queue 에서 CPU를 기다리며
스케줄러는 이 큐에서 다음 실행할 프로세스를 선택한다.

---

## 6. 타이머 인터럽트와 상태 변화

운영체제는 타이머 인터럽트를 사용해 CPU 사용 시간을 제한한다.

- 실행 중인 프로세스 시간 초과
- Running -> Ready 전환
- 다른 프로세스 실행
  이를 통해 멀티태스킹이 가능해진다

---

## 7. 상태 전이 다이어그램

New  
 ↓  
Ready ←→ Running  
 ↑ ↓  
Waiting  
 ↓  
Terminated

---

## 8. 확장 상태 (툰영체제에 따라 추가)

- Ready Suspended
- Blocked Suspended
  메모리 부족 등으로 디스크로 이동된 상태를 의미한다.

---

## 9. 프로세스 상태와 운영체제의 역할

운영체제는

- 상태 전이 관리
- Ready Queue 관리
- CPU 할당 결정
- 인터럽트 처리
  를 통해 전체 시스템을 조율한다

---

## 10. 정리

프로세스는

- 생성
- 준비
- 실행
- 대기
- 종료
  의 상태를 오가며 동작한다
