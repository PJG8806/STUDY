## 1. 인터럽트란 무엇인가

인터럽트(Interrupt)는 CPU가 현재 작업을 수행하는 도중에
즉시 처리해야 할 사건이 발생했음을 알리는 신호
인터럽트가 발생하면

1. CPU는 현재 작업을 잠시 멈춘다.
2. 인터럽트 처리 루틴을 실행한다.
3. 처리가 끝나면 원래 작업으로 복귀한다
   인터럽트는 지금 하고 있는 것 잠깐 멈추고 이것부터 처리 하라는 신호

---

## 2. 인터럽트가 필요한 이유

CPU는 매우 빠르게 동작하지만 입출력 장치는 상대적으로 느리며
만약 CPU가 장치 상태를 계속 확인한다면 (Polling)

- CPU 자원 낭비
- 비효율적인 동작
  문제가 발생으로 필요할 때만 CPU에 알림을 주어 효율적인 처리를 한다.

---

## 3. 인터럽트의 종류

### 3.1 하드웨어 인터럽트

외부 장치에 의해 발생한다.

- 키보드 입력
- 마우스 클릭
- 디스크 작업 완료
- 네트워크 데이터 도착

### 3.2 소프트웨어 인터럽트

프로그램 내부에서 발생한다.

- 시스템 콜 (System Call)
- 예외 (Exception)

### 3.3 예외 (Exception)

프로그램 실행 중 비정상 상황 발생 시 발생한다.

- 0으로 나누기
- 잘못된 메모리 접근

---

## 4. 인터럽트 처리 과정

1. 현재 실행 중인 명령어 완료
2. 현재 상태 저장 (제리스터, PC등)
3. 인터럽트 서비스 루틴(ISR) 실행
4. 처리 완료 후 상태 복원
5. 원래 작업 계속 수행

---

## 5. 인터럽트 백터

인터럽트가 발생하면 어떤 코드를 실행해야 하는지 알아야 하며
이를 위해 인터럽트 백터 테이블이 존재한다

- 인터럽트 번호
- 해당 처리 함수 주소
  를 연려해 둔 표이다

---

## 6. 인터럽트와 운영체제

운영체제는 인터럽트를 이용해 시스템을 제어한다.

- 타이머 인터럽트 -> CPU 스케줄링 수행
- I/O 인터럽트 -> 장치 작업 완료 처리
- 시스템 콜 -> 커널 모드 전환
  인터럽트는 운영체제가 CPU를 통제하는 핵심 수단이다.

---

## 7. 인터럽트와 컨택스트 스위칭

타이머 인터럽트가 발생하면 운영체제는 현재 프로세스를 멈추고
다른 프로세스로 전환할 수 있으며 이때 발생하는 과정이
컨텍스트 스위칭(Context Switching)이다.

---

## 8. 인터럽트와 폴링(Polling) 비교

| 구분      | 인터럽트          | 폴링            |
| --------- | ----------------- | --------------- |
| 동작 방식 | 장치가 CPU에 알림 | CPU가 계속 확인 |
| 효율성    | 높음              | 낮음            |
| CPU 낭비  | 적음              | 많음            |

---

## 9. 정리

인터럽트는 CPU가 외부 사건에 빠르게 반응하도록 만드는 메커니즘이다.

- 작업 중단
- 처리
- 복귀
  운영체제는 인터럽트를 통해 CPU, 장치, 프로세스를 통제한다.
