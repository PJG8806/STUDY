## 1. 컴퓨터 시스템의 기본 구성

1. CPU (중앙처리장치)
2. 메모리 (주기억장치)
3. 보조기억장치 (저장장치)
4. 입출력장치 (I/O 장치)

---

## 2. CPU (Central Processing Unit)

CPU는 컴퓨터의 두뇌 역할을 하며 프로그램의 명령어를 해석하고 실행하는 장치이다.

### 2.1 산술논리연산장치 (ALU)

- 덧셈, 뺄셈 등의 산술 연산
- AND, OR 같은 논리 연산 수행
  계산을 담당한다.

### 2.2 제어장치 (Control Unit)

- 명령어 해석
- 각 장치에 신호 전달
- 전체 시스템의 흐름 제어
  CPU의 지휘자 역할을 한다.

### 2.3 레지스터 (Register)

- CPU 내부의 매우 빠른 임시 저장 공산
- 명령어, 주소, 연산 결과 등을 저장
  레지스터는 메모리보다 빠르다

---

## 3. 메모리 구조

### 3.1 주기억장치 (RAM)

- 프로그램이 실행될 때 올라가는 공간
- 휘발성 메모리 (전원이 꺼지면 데이터 사라짐)
- CPU가 직접 접근 가능
  RAM이 크면 동시에 더 많은 프로그램 실행이 가능하다.

### 3.2 보조기억장치 (SSD / HDD)

- 데이터를 장기간 저장
- 비휘발성 메모리
- RAM보다 느리지만 저장 용량이 큼
  운영체제와 프로그램은 보조기억장치에 저장되어 있다가 실행 시 RAM으로 올린다.

---

## 4. 입출력 장치 (I/O Device)

### 입력 장치

- 키보드
- 마우스
- 카메라

### 출력장치

- 모니터
- 프린터
- 스피커

### 통신 장치

- 네트워크 카드

운영체제는 장치 드라이버를 통해 이 장치들을 제어한다.

---

## 5. 버스 (Bus)

버스는 컴퓨터 내부 부품들을 연결하는 통로이다.

### 5.1 데이터 버스

데이터를 전달

### 5.2 주소 버스

메모리 위치 지정

### 5.3 제어 버스

읽기/쓰기 등의 제어 신호 전달

CPU, 메모리, 장치들은 버스를 통해 서로 통신한다.

---

## 6. 명령어 처리 과정 (Fetch - Decode - Execute)

CPU는 다음 과정을 반복하며 동작한다

1. Fetch (인출)

- 메모리에서 명령어 가져오기

2. Decode (해독)

- 명령어 해석

3. Excute (실행)

- 연산 수행

---

## 7. 메모리 계층 구조

컴퓨터 메모리는 속도와 용량에 따라 계층 구조를 가진다

빠름 / 비쌈 / 작음

- 레지스터
- 캐시 메모리
- RAM

느림 / 저렴 / 큼

- SSD / HDD

CPU에 가까울수록 빠르고 비싸며 용량이 작다.

---

## 8. 캐시 메모리

- CPU와 RAM 사이의 위치
- 자주 사용하는 데이터를 임시 저장
- 속도 향상 목적
  캐시는 지역성 (Locality) 원리를 이용한다.

---

## 9. 전체 동작 흐름 요약

1. 프로그램이 SSD에 저장되저 있음
2. 실행하면 RAM으로 로드
3. CPU가 명령어를 하나씩 처리
4. 결과를 다시 메모리나 저장장치에 기록
   운영체제는 이 모든 과정을 관리한다.

---

## 10. 정리

- CPU -> 계산
- 메모리 -> 임시 저장
- 저장장치 -> 장기 저장
- I/O -> 외부와 통신
