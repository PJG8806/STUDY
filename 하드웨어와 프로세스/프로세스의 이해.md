## 1. 프로세스란 무엇인가

프로세스(Process)는 실행 중인 프로그램이며
디스크에 저장도니 상태는 프로그램(Program)이고,
메모리에 올라와 CPU에 의해 실행되는 순간부터 프로세스가 된다.

프로그램 = 정적인 파일
프로세스 = 동적인 실행 단위

---

## 2. 프로그램과 프로세스의 차이

| 구분 | 프로그램     | 프로세스        |
| ---- | ------------ | --------------- |
| 상태 | 저장된 파일  | 실행 중         |
| 위치 | 보조기억장치 | 주기억장치(RAM) |
| 성격 | 정적         | 동적            |

같은 프로그램을 여러 번 실행하면 여러 개의 프로세스가 생성된다.

---

## 3. 프로세스의 구성 요소

프로세스는 단순한 코드 덩어리가 아니다.

### 3.1 코드 영역 (Code)

- 실행할 명령어 저장
- 읽기 전용

### 3.2 데이터 영역 (Data)

- 전역 변수
- static 변수

### 3.3 힙 영역 (Heap)

- 동적 메모리 할당 영역
- 실행 중 크기 변화 가능

### 3.4 스택 영역 (Stack)

- 함수 호출 정보 저장
- 지역 변수 저장
- LIFO 구조

---

## 4. 프로세스 제어 블록 (PCB)

운영체제는 각 프로세스를 관리하기 위해
PCB(PROCESS CONTROL BLOCK)를 만든다.

PCB에 저장되는 정보

- 프로세스 ID (PID)
- 현재 상태
- 프로그램 카운터 (PC)
- 레지스터 값
- 스케줄링 정보
- 메모리 정보
  PCB는 프로세스의 신분증과 같다

---

## 5. 프로세스 상태

프로세스는 항상 실행되는 것이 아니라 여러 상태를 오간다.

### 5.1 생성(New)

프로세스가 생성되는 단계

### 5.2 준비(Ready)

CPU 할당을 기다리는 상태

### 5.3 실행(Running)

CPU를 할당받아 실행 중인 상태

### 5.4 대기(Waiting / Blocked)

입출력 작업 등을 기다리는 상태

### 5.3 종료(Terminated)

실행이 끝난 상태

---

## 6. 컨텍스트 스위칭

CPU는 한 번에 하나의 프로세스만 실행할 수 있다
다른 프로세스로 전환할 때

1. 현재 프로세스의 상태를 PCB에 저장
2. 다음 프로세스의 상태를 복원
3. 실행 재개
   이 과정을 컨텍스트 스위칭(Context Switching) 이라고 한다.

---

## 7. 프로세스와 자원

- CPU 시간
- 메모리 공간
- 파일 핸들
- 입출력 장치
  운영체제는 자원 충돌이 발생하지 않도록 각 프로세스를 독립적으로 관리한다.

---

## 8. 부모 프로세스와 자식 프로세스

일부 운영체제(UNIX, Linux)에서는 프로세스가 또 다른 프로세스를 생성할 수 있다.

- 부모 프로세스(Parent)
- 자식 프로세스(Child)
  이를 통해 프로그램은 병렬 작업을 수행할 수 있다.

---

## 9. 프로세스와 스레드의 관계

프로세스는 독립적인 실행 단위이며 스레드는 프로세스 내부의 실행 흐름이다.

프로세스

- 독립 메모리
- 자원 분리

스레드

- 메모리 공유
- 가벼운 실행 단위

---

## 10. 정리

프로세스

- 실행 중인 프로그램
- 독립된 메모리 공간을 가지며
- 운영체제가 PCB로 관리하는 실행 단위
