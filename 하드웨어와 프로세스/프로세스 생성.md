## 1. 프로세스 생성이란 무엇인가

프로세스 생성은 운영체제가 새로운 실행 단위를 만드는 과정이며
사용자가 프로그램을 실행하거나 실행 중인 프로세스가 또 다른 프로세스를 만들 때
새로운 프로세스가 생성된다.

---

## 2. 언제 프로세스가 생성되는가

1. 사용자가 프로그램 실행
2. 다른 프로세스가 새로운 프로세스 생성
3. 시스템 서비스 실행
4. 로그인 시 사용자 세션 시작

---

## 3. 프로세스 생성 과정

### 3.1 실행 요청

- 사용자가 실행 명령 입력
- 운영체제가 실행 파일 위치 확인
- 실행 권한 검사

### 3.2 메모리 할당

운영체제는 새 프로세스를 위해

- 코드 영역
- 데이터 영역
- 힙 영역
- 스택 영역
  을 메모리에 할당한다.

### 3.3 PCB 생성

운영체제는 프로세스 제어 블록(PCB)를 생성한다

- PID (프로세스 ID)
- 현재 상태
- 프로그램 카운터
- 레지스터 값
- 스케줄링 정보
- 메모리 정보
  PCB는 프로세스를 관리하는 핵심 자료구조이다.

### 3.4 초기 상태 설정

- 상태를 Ready로 설정
- Ready Queue에 삽입
- CPU 할당 대기

---

## 4. 부모 프로세스와 자식 프로세스

일부 운영체제(UNIX 계열)에서는 프로세스가 또 다른 프로세스를 생성할 수 있다.

- 부모 프로세스 (Parent)
- 자식 프로세스 (Child)
  자식 프로세스는 부모 프로세스의 일부 자원을 상속받을 수 있다.

---

## 5. fork()와 exec() (UNIX 기준 개념)

### 5.1 fork()

- 현재 프로세스를 복사하여 새로운 프로세스 생성
- 부모와 거의 동일한 자식 프로세스 생성

### 5.2 exec()

- 현재 프로세스의 메모리를 새로운 프로그램으로 교체
- 보통 fork() 이후 호출

fork() → 복제  
exec() → 새로운 프로그램 실행

---

## 6. 프로세스 생성 시 자원 할당

새 프로세스에는 다음이 할당된다.

- 고유 PID
- 메모리 공간
- 파일 디스크립터
- CPU 사용 권한
  운영체제는 자원 부족 시 프로세스 생성을 거부할 수 있다.

---

## 7. 프로세스 생성의 비용

프로세스를 생성하는 것은 비용이 크다.

- 메모리 할당
- PCB 생성
- 자원 초기화
- 컨텍스트 관리

그래서 경량 실행 단위인  
스레드가 등장하게 되었다.

---

## 8. 프로세스 생성 후 흐름

1. 생성(New)
2. 준비(Ready)
3. 실행(Running)
4. 대기(Waiting)
5. 종료(Terminated)
   생성은 전체 실행 과정의 시작 단계이다.

---

## 9. 정리

프로세스 생성은

- 실행 요청
- 메모리 할당
- PCB 생성
- Ready 상태 진입
  을 거치며

운영체제는 각 프로세스를 독립적으로 관리하며 시스템 자원을 효율적으로 배분한다.
