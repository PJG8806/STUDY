## 1. 프로그램과 프로세스의 차이

- 프로그램(Program): 디스크에 저장된 실행 파일
- 프로세스(Process): 실행 중인 프로그램
  디스크에 있는 상태 -> 프로그램
  메모리에 올라와 실행 중인 상태 -> 프로세스

---

## 2. 1단계: 프로그램 실행 요청

사용자가 실행 파일을 더블 클릭하거나 터미널에서 실행 명령을 입력한다.
이때 운영체제가 개입한다.

운영체제

- 실행 파일 위치 확인
- 실행 권한 확인
- 메모리 공간 확보

---

## 3. 2단계: 메모리 로드(Loading)

운영체제는 실행 파일을 보조기억장치(SSD/HDD)에서 주기억장치(RAM)로 복사한다.

메모리

- 코드 영역 (Code)
- 데이터 영역 (Data)
- 힙 영역 (Heap)
- 스택 영역 (Stack)
  각 영역은 서로 다른 역할을 가진다

---

## 4. 프로세스 메모리 구조

### 4.1 코드 영역

- 실행할 명령어 저장
- 읽기 전용

### 4.2 데이터 영역

- 전역 변수, static 변수 저장

### 4.3 힙 영역

- 동적 메모리 할당 영역
- 프로그램 실행 중 크기 변동 가능

### 4.4 스택 영역

- 함수 호출 정보 저장
- 지역 변수 저장
- LIFO 구조

---

## 5. 3단계: 프로세스 생성

메모리에 로드가 완료되면 운영체제는 프로세스를 생성한다

생성되는 것:

- 프로세스 ID (PID)
- 프로세스 제어 블록 (PCB)
- 프로세스 상태 정보

PCB에는 다음 정보가 저장

- 현재 상태
- 프로그램 카운터 (PC)
- 레지스터 값
- 메모리 정보

---

## 6. 4단계: CPU 할당 (스케줄링)

CPU는 한 번에 하나의 작업만 처리할 수 있으며 운영체제는 여러 프로세스 중
어떤 프로세스에 CPU를 줄지 결정한다
이 과정을 스케줄링이라고 하며 선택된 프로세스는 실행 상태가 된다.

---

## 7. 5단계: 명령어 실행 (Fetch - Decode - Execute)

1. Fetch
   메모리에서 명령어를 가져온다
2. Decode
   명령어 해석
3. Execute
   연산 수행 또는 작업 처리

---

## 8. 실행 중 발생하는 일

### 8.1 시스템 콜

프로그램이 파일을 열거나 네트워크를 사용하려면 운영체제에게 요청해야 하며 이를 시스템 콜이라고 한다.

### 8.2 인터럽트

키보드 입력, 마우스 클릭, 디스크 완료 등 외부 신호가 들어오면 CPU는 현재 작업을
잠시 멈추고 해당 작업을 처리한다.

### 8.3 컨텍스트 스위칭

CPU가 다른 프로세스로 전환할 때

- 현재 상태 저장
- 다른 프로세스 상태 복원
  이 과정을 컨텍스트 스위칭이라고 한다.

---

## 9. 6단계: 프로그램 종료

프로그램이 실행을 마치면

- 사용한 메모리 해제
- PCB 제거
- 자원 반환
  운영체제는 시스템을 깨긌한 상태로 유지한다.

---

## 10. 전체 흐름 정리

1. 실행 요청
2. 디스크 -> RAM 로드
3. 프로세스 생성
4. CPU 할당
5. 명령어 실행 반복
6. 종료 및 지원 반환

---

# 11. 핵심 이해 포인트

- 프로그램은 디스크에 있고, 실행되면 메모리에 올라온다.
- CPU는 명령어를 한 줄로 처리한다.
- 운영체제가 전체 과정을 관리한다.
- 실행 과정은 하드웨어와 운영체제가 협력하는 결과이다.
