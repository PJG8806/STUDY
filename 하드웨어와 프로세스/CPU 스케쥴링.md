## 1. CPU 스케줄링이란 무엇인가

CPU 스케줄링(CPU Scheduling)은 여러 프로세스 중에서 어떤 프로세스에게
CPU를 할당할지 결정하는 과정이며 CPU는 한 번에 하나의 작업만 수행할 수 있다
운영체제는 스케줄러(Scheduler)를 통해 CPU 사용 순서를 결정한다.

---

## 2. 왜 CPU 스케줄링이 필요한가

여러 프로세스가 동시에 Ready 상태에 있을 때:

- 누구를 먼저 실행할 것인가
- 얼마나 오래 실행시킬 것인가
- 공정하게 배분할 것인가
  등의 문제를 해결하기 위해 CPU 스케줄링이 필요하다.

---

## 3. 스케줄링의 목표

좋은 스케줄링은 다음을 고려한다.

- CPU 이용률 증가
- 응답 시간 최소화
- 대기 시간 최소화
- 처리량(Throughput) 증가
- 공정성 유지

---

## 4. 선점형 vs 비선점형 스케줄링

### 4.1 비선점형 (Non-preemptive)

한 프로세스가 CPU를 사용하면 스스로 종료하거나 대기 상태가 될 때까지 계속 사용

### 4.2 선점형 (Preemptive)

- 운영체제가 강제로 CPU를 빼앗을 수 있음
- 타이머 인터럽트 사용

---

## 5. 주요 스케줄링 알고리즘

### 5.1 FCFS (First Come First Served)

먼저 도착한 프로세스를 먼저 실행
장점

- 구현 간단
  단점
- 긴 작업이 앞에 있으면 대기 시간 증가
- Convoy Effect 발생 가능

### 5.2 SJF (Shortest Job First)

실행 시간이 가장 짧은 프로세스를 먼저 실행
장점

- 평균 대기 시간 최소화
  단점
- 실행 시간을 정확히 알기 어려움
- 긴 작업이 계속 밀릴 수 있음

### 5.3 Priority Scheduling

우선순위가 높은 프로세스를 먼저 실행
장점

- 중요한 작업 먼저 처리 가능
  단점
- 낮은 우선순위 프로세스가 무한 대기할 수 있음 (기아 현상)

### 5.4 Round Robin (RR)

각 프로세스에게 일정 시간(Time Quantum)씩 CPU를 할당

- 선점형
- 공정한 분배
  Time Quantum이 너무 크면 FCFS와 유사  
  너무 작으면 컨텍스트 스위칭 증가

---

## 6. 스케줄링과 컨텍스트 스위칭

스케줄러가 다른 프로세스를 선택하면 컨텍스트 스위칭이 발생한다.
스케줄링 = 누구를 실행할지 결정  
컨텍스트 스위칭 = 실제 전환 과정

---

## 7. 스케줄링 관련 주요 용어

- 대기 시간 (Waiting Time)
- 응답 시간 (Response Time)
- 반환 시간 (Turnaround Time)
- 처리량 (Throughput)
- CPU 이용률 (CPU Utilization)

## 8. 실제 운영체제의 스케줄링

현대 운영체제는 단일 알고리즘이 아닌 복합적인 방식 사용
목표는

- 인터랙티브 작업은 빠르게
- 배치 작업은 효율적으로
- 전체 시스템 성능 최적화

---

## 9. 정리

CPU 스케줄링

- 여러 프로세스 중 CPU를 누구에게 줄지 결정하는 과정
- 선점형과 비선점형으로 구분
- 다양한 알고리즘 존재

스케줄링은 운영체제가 CPU를 효율적으로 관리하는 핵심 기능이다.
