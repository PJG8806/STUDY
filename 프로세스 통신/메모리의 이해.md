## 1. 메모리란 무엇인가

메모리(Memory)는 CPU가 작업을 수행하기 위해 사용하는 데이터 저장 공간이며
프로그램은 실행되기 위해 반드시 메모리에 올라가야 한다
CPU는 디스크가 아니라 메모리에 있는 데이터를 직접 처리한다.
디스크 = 장기 저장  
메모리(RAM) = 실행 공간

---

## 2. 메모리가 필요한 이유

CPU는 매우 빠르지만 보조기억장치(SSD, HDD)는 상대적으로 느리다.

만약 CPU가 디스크에서 직접 데이터를 읽는다면 전체 시스템 성능이 크게 떨어진다.

- 프로그램을 메모리로 불러오고
- 메모리에서 연산을 수행한다.

---

## 3. 메모리의 종류

### 3.1 주기억장치 (RAM)

- 실행 중인 프로그램 저장
- 휘발성 (전원 차단 시 데이터 사라짐)
- CPU가 직접 접근 가능

### 3.2 보조기억장치 (SSD / HDD)

- 프로그램과 데이터 장기 저장
- 비휘발성
- RAM보다 느림

### 3.3 캐시 메모리 (Cache)

- CPU와 RAM 사이에 위치
- 자주 사용하는 데이터 임시 저장
- 매우 빠름

---

## 4. 메모리 계층 구조

메모리는 속도와 용량에 따라 계층 구조를 가진다.

위로 갈수록

- 빠름
- 비쌈
- 용량 작음

아래로 갈수록

- 느림
- 저렴
- 용량 큼

순서
레지스터  
→ 캐시  
→ RAM  
→ SSD/HDD

---

## 5. 프로세스의 메모리 구조

### 5.1 코드 영역 (Text Segment)

- 실행할 명령어 저장
- 읽기 전용

### 5.2 데이터 영역 (Data Segment)

- 전역 변수
- static 변수

### 5.3 힙 영역 (Heap)

- 동적 메모리 할당
- 실행 중 크기 변동 가능

### 5.4 스택 영역 (Stack)

- 함수 호출 정보 저장
- 지역 변수 저장
- LIFO 구조

---

## 6. 메모리 관리의 목적

운영체제는 다음을 위해 메모리를 관리한다.

1. 효율적 사용
2. 보호(Protection)
3. 공유(Sharing)
4. 확장성
   각 프로세스는 서로의 메모리에 접근할 수 없다.

---

## 7. 가상 메모리

가상 메모리는 실제 메모리보다 더 큰 공간을 사용하는 것처럼 보이게 하는 기술이다.
원리

- RAM이 부족하면
- 디스크 일부를 임시 메모리처럼 사용

이를 통해

- 큰 프로그램 실행 가능
- 여러 프로그램 동시 실행 가능

---

## 8. 페이징(Paging)

메모리를 일정한 크기의 블록(페이지)으로 나누는 방식이다.
장점

- 외부 단편화 감소
- 효율적인 관리

운영체제는 페이지 테이블을 통해 가상 주소를 물리 주소로 변환한다.

---

## 9. 메모리 단편화

### 내부 단편화

할당된 공간 내부에서 낭비 발생

### 외부 단편화

사용 가능한 공간이 흩어져 있음

운영체제는 이를 줄이기 위해 페이징이나 세그멘테이션을 사용한다.

---

## 10. 메모리와 CPU의 관계

CPU는 직접 메모리에 접근하여 데이터를 가져온다.
과정

1. 주소 지정
2. 데이터 읽기
3. 레지스터에 저장
4. 연산 수행
   메모리 속도는 전체 시스템 성능에 큰 영향을 준다.

---

## 11. 정리

메모리는

- 프로그램 실행 공간
- CPU 작업을 위한 데이터 저장소
- 운영체제가 관리하는 핵심 자원
