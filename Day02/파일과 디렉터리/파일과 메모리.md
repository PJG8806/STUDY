## 1. 파일과 메모리

운영체제에서 메모리(Memory)의 파일(File)은 각각 작업하는 책상과 보관하는 서랍으로 비유할 수 있다

- 메모리 (책상): 지금 당장 필요한 서류를 올려두고 작업하는 좁지만 빠른 공간
- 파일 (서랍): 당장 쓰지 않는 서류를 보관하는 넓지만 꺼내기 느린 공간

---

## 2. 주요 특징 비교

- 목적
  메모리는 현재 실행 중인 프로그램과 데이터의 작업 공간
  파일은 데이터와 프로그램의 영구적인 보관 및 공유
- 휘발성
  메모리는 휘발성 (전원이 꺼지면 사라짐)
  파일은 비휘발성 (전원이 꺼져도 유지)
- 접근 속도
  메모리가 매우 빠름
  파일이 상대적으로 느림
- CPU 접근
  메모리는 CPU가 직접 접근 가능
  파일은 직접 접근 불가 (반드시 메모리로 불러와야 함)
- 관리 단위
  메모리는 페이지, 세그먼트
  파일은 블록, 섹터

---

## 3. 파일과 메모리의 상호작용

CPU는 파일(디스크)에 있는 데이터를 직접 연산할 수 없으며 둘 사이에는 끊임없는 데이터 이동이 발생합니다.

- 프로그램의 실행 (Load): 하드디스크에 저장된 파일(프로그램)을 실행하면 운영체제는 이 내용을 메모리로 적재(Load)합니다.
- 스왑 공간 (Swap Space): 메모리가 꽉 찼을 때 하드디스크의 일정 영역을 메모리처럼 사용하며 디스크 메모리의 연장선 역할을 합니다.
- 메모리 맵 파일 (Memory-Mapped File): 파일 입출력 속도를 높이는 기술로 파일의 데이터를 메모리의 특정 주소 공간으로 바로 매핑합니다.

---

## 4. 핵심 요약

- 메모리는 실행을 위해 파일은 보관을 위해 존재한다.
- 어떤 파일이든 가공되거나 실행되려면 반드시 메모리에 올려야 한다.
- 현대 운영체제는 가상 메모리와 메모리 맵 파일 기술을 통해 메모리와 파일의 경계를 허물며 시스템을 최적화하고 있다.
