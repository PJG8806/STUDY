## 1. 디렉터리(Directory)란 무엇인가

디렉터리는 수많은 파일들을 효율적으로 구성하고 관리하기 위한 논리적인 그룹화 방식이며
우리가 흔히 부르는 폴더(Folder)와 같은 개념이며 운영체제 입장에서는 디렉터리 역시
다른 파일들의 목록과 위치 정보를 담고 있는 특수한 형태의 파일로 취급합니다.

---

## 2. 디렉터리의 핵심 역할

- 파일 탐색: 파일 이름을 통해 디스크 상의 실제 물리적 주소를 찾아냅니다.
- 파일 관리: 파일의 생성, 삭제, 이름 변경, 목록 조회 등의 작업을 돕습니다.
- 이름 공간(Namespace) 분리: 서로 다른 디렉터리에 있다면 동일한 이름의 파일을 여러 개 만들 수 있도록 해줍니다.

---

## 3. 디렉터리의 논리적 구조 발전 과정

- 단일 단계 (Single-Level): 모든 파일이 하나의 디렉터리에 존재 하며 이름 충돌 발생, 관리 어려움
- 2단계 (Two-Level): 사용자 마다 별도의 디렉터리를 가지며 사용자 간 충돌은 막지만 그룹화 불가
- 트리 구조 (Tree-Structured): 하나의 루트에서 시작해 가지를 치는 계층 구조
- 비순환 그래프 (Acyclic-Graph): 트리 구조에 공유(링크, 바로가기) 개념이 추가된 구조
- 일반 그래프 (General Graph): 사이클(순환 참조)을 허용하는 구조로 무한 루프 위험이 있다

---

## 4. 경로 (Path)의 개념

- 절대 경로 (Absolute Path): 최상위 루트 디렉터리부터 목표 파일까지의 전체 경로
- 상대 경로 (Relative Path): 현재 작업 중인 디렉터리(Current Working Directory)를 기준으로 목표 파일까지의 경로

---

## 5. 디렉터리 구현 방식 (OS 내부)

- 선형 리스트 (Linear List): 파일 이름과 포인터를 순차적으로 저장하며 구현은 쉽지만 탐색 속도가 느림
- 해시 테이블 (Hash Table): 파일 이름을 해시 함수에 넣어 위치를 즉시 찾아내며 탐색 속도는 빠르나 해시 충돌 관리가 필요함

---

## 6. 핵심 요약

- 디렉터리는 파일들을 효율적으로 찾고 묶어두기 위한 색인(Index) 파일이다.
- 현대 운영체제는 대부분 트리 구조(Tree) 기반에 비순환 그래프(공유 링크) 특성을 결합하여 사용한다.
- 파일을 지칭할 때는 루트 기준의 절대 경로와 현재 위치 기준의 상대 경로를 명확히 구분해야 한다.
