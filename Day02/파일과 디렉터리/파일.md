## 1. 파일(File)이란 무엇인가

파일은 운영체제가 관리하는 논리적인 정보 저장의 기본 단위이며 사용자가 하드디스크나
SSD의 물리적 구조(섹터, 트랙)를 알 필요 없이 데이터를 하나의 덩어리로 다룰 수 있게 해주는 추상화된 개념

---

## 2. 파일이 필요한 이유

- 비휘발성 (Persistence): 전원이 꺼져도 데이터를 영구적으로 보존해야 한다 (메모리와의 가장 큰 차이)
- 대용량 저장: 주 기억장치(RAM)의 한정된 용량을 극복하고 대량의 데이터를 저장한다.
- 공유 (Sharing): 여러 사용자나 프로세스가 동일한 데이터에 접근하고 사용할 수 있게 한다.

---

## 3. 파일의 속성과 연산

### 3.1 파일 속성 (Metadata)

파일 자체의 내용(Data)이 아니라 파일을 관리하기 위해 운영체제가 유지하는 파일에 대한 정보

- 이름 (Name), 식별자 (Identifier)
- 유형 (Location, 디스크 상의 주소)
- 크기 (Size)
- 보호 (Protection, 접근 권한 - 읽기/쓰기/실행) -시간, 날짜, 사용자 식별 정보 (생성/최근 수정/최근 접근)

### 3.2 파일 연산 (File Operations)

운영체제가 시스템 콜(System Call)을 통해 제공하는 파일 제어 기능

- 생성 (Create), 쓰기 (Write), 읽기(Read), 위치 재설정 (Reposition, Seek), 삭제 (Delete),
  자르기 (Truncate)
- 열기 (Open) / 닫기 (Close): 파일 연산을 수행하기 전후에 반드시 거쳐야 하는 과정으로 운영체제 메모리 내의
  열린 파일 테이블(Open-file table)을 관리합니다.

---

## 4. 파일 할당 방법 (Allocation Methods)

디스크의 빈 공간에 파일 데이터를 어떻게 배치할 것인가를 결정하는 방식

- 연속 할당 (Contiguous)
  디스크의 연속된 블록에 파일 저장
  순차/직접 접근 모두 빠름
  외부 단편화 발생
  파일 크기 확정 어려움
- 연결 할당 (Linked)
  파일 데이터를 여러 블록에 분산시키고 포인터로 연결
  외부 단편화 없음
  직접 접근 (Random Access) 불가
  포인터 공간 낭비
  신뢰성 문제
- 색인 할당 (Indexed)
  모든 포인터를 모아둔 하나의 인덱스 블록 사용
  직접 접근 가능
  외부 단편화 없음
  인덱스 블록을 위한 추가 공간 필요

---

## 5. 파일 시스템의 종류

- Windows: FAT32 (호환성 높음), NTFS (보안, 대용량 지원)
- Linux/UNIX: ext2, ext3, ext4 (저널링 기능 강화)
- 가상 파일 시스템 (VFS, Virtual File System): 운영체제가 서로 다른 여러 종류의 파일 시스템(NTFS, ext4)을
  동일한 방식(인터페이스)으로 접근할 수 있게 해주는 추상화 계층

## 6. 주요 개념 요약

- 파일 시스템의 핵심 목표
  데이터를 디스크에 어떻게 저장하는지
  어떻게 빨리 찾을 것인가를 해결하는 것
- 메모리 VS 파일
  메모리는 프로세스의 작업 공간(휘발성)
  파일은 데이터의 창고(비휘발성)
- 단편화
  디스크 공간이 조각나는 현상
  파일 시스템의 성능을 저하시키는 주범
