## 1. 시나리오 설정

- 물리 메모리 프레임(Frame) 개수: 3개
- 페이지 참조열 (Reference String): [7, 0, 1, 2, 0, 3, 0, 4]

---

## 2. FIFO (First-In First-Out) 동작 과정

규칙: 메모리에 가장 먼저 들어온 페이지를 쫒아냅니다.

- 7 요청: [7, -, -] Fault
- 0 요청: [7, 0, -] Fault
- 1 요청: [7, 0, 1] Fault
- 2 요청: 가장 먼저 들어온 7 교체 -> [2, 0, 1] Fault
- 0 요청: 이미 있음 -> [2, 0, 1] Hit
- 3 요청: 다음으로 오래된 0 교체 -> [2, 3, 1] Fault
- 0 요청: 다음으로 오래된 1 교체 -> [2, 3, 0] Fault
- 4 요청: 다음으로 오래된 2 교체 -> [4, 3, 0] Fault

- 총 Fault: 7회

---

## 3. LRU (Least Recently Used) 동작 과정

규칙: 메모리에서 "가장 오랫동안 사용되지 않은" 페이지를 쫓아냅니다.

- 7 요청: [7, -, -] Fault
- 0 요청: [7, 0, -] Fault
- 1 요청: [7, 0, 1] Fault
- 2 요청: 가장 예전에 쓰인 7 교체 -> [2, 0, 1] Fault
- 0 요청: 이미 있음 (최근 사용으로 갱신!) -> [2, 0, 1] Hit
- 3 요청: 가장 예전에 쓰인 1 교체 -> [2, 0, 3] Fault
- 0 요청: 이미 있음 -> [2, 0, 3] Hit
- 4 요청: 가장 예전에 쓰인 2 교체 -> [4, 0, 3] Fault

- 총 Fault: 6회

---

## 4. 핵심 결과 분석

FIFO는 방금 사용된 데이터라도 들어온 지 오래되었다면 무조건 쫓아냅니다.
반면 LRU는 방금 사용된 데이터(0)를 기억하고 생명을 연장해 주어 지역성(Locality)을 잘 활용합니다.
따라서 LRU가 더 효율적입니다.
