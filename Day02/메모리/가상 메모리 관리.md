## 1. 가상 메모리란 무엇인가

가상 메모리(Virtual Memory)는 실제 물리 메모리(RAM)보다 더 큰 메모리를 사용하는 것처럼 보이게 하는 기술이다.
프로그램은 자신이 큰 연속된 메모리 공간을 사용하는 것처럼 보이지만 실제로는 물리 메모리가 분산되어 사용된다.

---

## 2. 왜 가상 메모리가 필요한가

물리 메모리(RAM)는 용량이 제한적이다.
문제

- 여러 프로그램이 동시에 실행됨
- 큰 프로그램 실행 필요
- 메모리 보호 필요

해결

- 가상 주소 공간 제공
- 필요한 부분만 RAM에 적재
- 나머지는 디스크에 저장

---

## 3. 가상 주소와 물리 주소

### 가상 주소 (Virtual Address)

- 프로그램이 사용하는 주소
- 각 프로세스마다 독립적

### 물리 주소 (Physical Address)

- 실제 RAM의 주소

운영체제와 하드웨어(MMU)가 가상 주소를 물리 주소로 변환한다.

---

## 4. MMU (Memory Management Unit)

MMU는 CPU 내부에 있는 장치로

- 가상 주소 → 물리 주소 변환
- 주소 보호 기능 수행
  페이지 테이블을 참조하여 변환한다.

---

## 5. 페이징(Paging)

가상 메모리를 일정한 크기의 블록으로 나누는 방식이다.

- 가상 메모리 → 페이지(Page)
- 물리 메모리 → 프레임(Frame)

---

## 6. 페이지 테이블(Page Table)

페이지 테이블은 가상 페이지와 물리 프레임을 연결하는 표이다.

- 물리 프레임 번호
- 유효 비트 (Valid Bit)
- 수정 비트 (Dirty Bit)
- 접근 비트 (Access Bit)

---

## 7. 페이지 폴트(Page Fault)

프로그램이 접근한 페이지가 RAM에 없는 경우 발생한다.
처리 과정

1. 페이지 풀트 발생
2. 운영체제가 디스크에서 해당 페이지 읽기
3. RAM에 적재
4. 페이지 테이블 갱신
5. 명령어 재실행

---

## 8. 페이지 교체(Page Replacement)

RAM이 가득 찬 상태에서 새로운 페이지가 필요할 경우 기존 페이지를 제거해야 하며
이를 페이지 교체라고 한다.

---

## 9. 대표적인 페이지 교체 알고리즘

### 9.1 FIFO

먼저 들어온 페이지를 먼저 제거
단점

- 성능이 좋지 않을 수 있음

### 9.2 LRU (Least Recently Used)

가장 오랫동안 사용되지 않은 페이지 제거
장점

- 지역성(Locality) 원리 반영

### 9.3 Optimal

앞으로 가장 오래 사용되지 않을 페이지 제거
이론적으로 가장 좋지만 미래를 알 수 없으므로 실제 구현 어려움

---

## 10. 지역성 (Locality)

프로그램은 다음 특성을 가진다.

### 시간적 지역성

최근 사용한 데이터는 곧 다시 사용될 가능성 높음

### 공간적 지역성

어떤 데이터를 사용하면 그 주변 데이터도 사용할 가능성 높음

가상 메모리 관리 기법은 이 지역성 원리를 활용한다.

---

## 11. 가상 메모리의 장점

- 큰 프로그램 실행 가능
- 메모리 보호 강화
- 멀티프로그램 환경 지원
- 메모리 효율 향상

---

## 12. 가상 메모리의 단점

- 페이지 폴트 발생 시 성능 저하
- 디스크 접근 비용 큼
- 과도한 페이지 교체 시 Thrashing 발생 가능

---

## 13. 정리

가상 메모리는

- 가상 주소 공간 제공
- 페이징을 통해 관리
- 페이지 교체 알고리즘 사용
- MMU와 운영체제가 협력하여 동작
  하는 메모리 관리 기법이다.
